# AutoFlow Infrastructure

AWS ECS Fargate deployment with GitHub Actions CI/CD.

## Structure

```
infra/
├── terraform/
│   ├── modules/
│   │   ├── networking/   VPC, subnets, NAT gateways
│   │   ├── ecr/          Container registries
│   │   ├── alb/          Application load balancer + target groups
│   │   ├── ecs/          ECS cluster, services, autoscaling
│   │   └── iam/          GitHub Actions OIDC role
│   └── environments/
│       ├── staging/      staging.autoflow.app
│       └── production/   autoflow.app
├── monitoring/
│   └── cloudwatch-alarms.tf
└── ecs/
    └── task-definitions/  (generated by CI, not committed)
```

## Bootstrap (first time)

1. Create S3 bucket + DynamoDB table for Terraform state:
   ```bash
   aws s3 mb s3://autoflow-terraform-state --region us-east-1
   aws dynamodb create-table \
     --table-name autoflow-terraform-locks \
     --attribute-definitions AttributeName=LockID,AttributeType=S \
     --key-schema AttributeName=LockID,KeyType=HASH \
     --billing-mode PAY_PER_REQUEST
   ```

2. Apply OIDC IAM role (one-time):
   ```bash
   cd infra/terraform/modules/iam
   terraform init && terraform apply
   ```

3. Add GitHub Actions secrets:
   - `AWS_ACCOUNT_ID`

4. Configure GitHub environment protection rules:
   - `staging` — no approval needed
   - `production` — require manual approval from team leads

5. Deploy staging:
   ```bash
   cd infra/terraform/environments/staging
   terraform init
   terraform apply -var="certificate_arn=arn:aws:acm:..." -var="ops_email=ops@autoflow.app"
   ```

## Daily operations

- Deploy to staging: merge to `main`
- Promote to production: approve in GitHub Actions after staging smoke tests pass
- View logs: `aws logs tail /ecs/autoflow-staging/backend --follow`
- Scale manually: `aws ecs update-service --cluster autoflow-staging --service autoflow-backend-staging --desired-count 3`

## Secrets

Secrets live in AWS Secrets Manager at `autoflow/{env}/app`. Update via:
```bash
aws secretsmanager put-secret-value \
  --secret-id autoflow/staging/app \
  --secret-string '{"DATABASE_URL":"...","SECRET_KEY":"...","REDIS_URL":"..."}'
```

ECS tasks fetch secrets at startup via IAM — no secrets in environment variables or code.
