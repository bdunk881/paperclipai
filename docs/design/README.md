# AutoFlow design references

The canonical design source for the AutoFlow product UI. **Reference only** — these files are HTML/CSS/JS prototypes built with Babel-in-browser React. Production implementations match their visual output; they don't import from this folder directly.

## v2 vs. legacy

| | v2 (canonical) | legacy (reference) |
|---|---|---|
| **Theme** | Cream paper + ink, terracotta accent, sage / mustard / plum | Indigo / teal / cyber-orange on obsidian dark |
| **Metaphor** | Workplace — workspaces as companies, agents as hires, missions as work | Workflow builder — nodes, edges, runs |
| **Type** | Fraunces serif + Geist + JetBrains Mono | Inter + JetBrains Mono |
| **IA** | Run · Workforce · Build · Connect | Templates · Builder · Runs · Marketplace · MCP |

The user explicitly directed the v2 redesign with *"Do not use our current theme. I want you to create something brand new."* — so v2 is the canonical direction. The legacy bundle is preserved because the workflow-builder details (cost log, prompt-to-flow shortcut, MCP server detail) are denser there and worth referencing when porting to production.

## How to view

These prototypes use `<script type="text/babel" src="...jsx">`, which browsers refuse to load over `file://`. Serve them over HTTP:

```bash
# From the repo root
npx --yes serve@14 --no-clipboard --no-port-switching -l 5174 docs/design
```

Then open <http://localhost:5174/v2/AutoFlow%20Landing.html> or <http://localhost:5174/v2/AutoFlow%20App.html>.

## How to use these in production work

When implementing UI work that touches the v2 redesign:

1. **Lift design tokens** from [`v2/styles.css`](v2/styles.css) lines 1–55 (the `:root` block and the `[data-af2-theme="dark"]` block) into `dashboard/tailwind.config.js`. The `--af2-*` custom properties are the source of truth — paper, ink, clay, sage, mustard, plum, ink-blue, plus the dark variant.
2. **Add the fonts** — Fraunces (serif display) + Geist (UI sans) + JetBrains Mono (code).
3. **Match the visual output** of the prototype HTMLs. Don't copy the prototype's internal structure unless it happens to fit — the JSX components are flat functions designed for Babel-in-browser, not production React.
4. **Don't ship the tweaks panel** (`v2/tweaks-panel.jsx`) — it's an internal design exploration tool.
